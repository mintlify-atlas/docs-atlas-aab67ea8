## Overview

The `CampaignsTable` component displays historical campaigns with:

1. Campaign date and total assigned profiles
2. Success/failure status indicators
3. Automatic data refresh when base changes
4. Error handling and loading states

Located at: `components/campaigns-table.tsx`

## Features

### 1. Fetch Campaigns from Database

Automatically loads campaigns when `baseId` is available:

```tsx
useEffect(() => {
  if (!isLoadingBase && baseId) {
    fetchCampaigns()
  }
}, [baseId, isLoadingBase])

const fetchCampaigns = async () => {
  const supabaseWithContext = createSupabaseClientWithContext(baseId)
  
  const { data, error } = await supabaseWithContext
    .from('campaigns')
    .select('campaign_id, campaign_date, total_assigned, status, created_at')
    .eq('base_id', baseId)
    .order('campaign_date', { ascending: false })
  
  setCampaigns(data || [])
}
```

### 2. Status Indicators

Visual status dots with color coding:

```tsx
const getStatusDot = (status: boolean) => {
  const statusColor = status ? "bg-green-500" : "bg-red-500"
  const statusText = status ? "success" : "failed"
  
  return (
    <div className="flex items-center gap-2">
      <div className={`h-2 w-2 rounded-full ${statusColor}`} />
      <span className="capitalize">{statusText}</span>
    </div>
  )
}
```

- **Green dot** - Distribution completed successfully to all VA tables
- **Red dot** - Distribution encountered errors

## Database Schema

### campaigns Table

```sql
CREATE TABLE campaigns (
  campaign_id UUID PRIMARY KEY,
  base_id UUID NOT NULL REFERENCES scraping_jobs(base_id),
  campaign_date DATE NOT NULL,
  total_assigned INTEGER NOT NULL,
  status BOOLEAN NOT NULL DEFAULT true,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

## Component Lifecycle

1. **Mount** - Component loads with initial loading state
2. **BaseContext resolves** - `baseId` becomes available
3. **Fetch data** - Query campaigns from Supabase
4. **Display results** - Show table or error/empty state
5. **BaseId changes** - State resets and re-fetches
6. **Unmount** - Cleanup via `usePageReset`
