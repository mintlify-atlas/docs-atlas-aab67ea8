## Overview

VA Distribution is the algorithm that evenly distributes scraped Instagram profiles across 80 Virtual Assistant (VA) tables, with each VA receiving a configurable number of profiles (default: 180).

<Info>
**Standard Configuration**: 80 VA tables × 180 profiles = 14,400 total profiles per campaign
</Info>

## Distribution Algorithm

The distribution uses a round-robin algorithm to ensure even distribution:

```javascript
function distributeProfiles(profiles, numVAs, profilesPerTable) {
  const assignments = [];
  
  profiles.forEach((profile, index) => {
    // VA table number: cycles 1-80 repeatedly
    const vaTableNumber = (index % numVAs) + 1;
    
    // Position within that VA's table
    const position = Math.floor(index / numVAs) + 1;
    
    assignments.push({
      ...profile,
      va_table_number: vaTableNumber,
      position: position
    });
  });
  
  return assignments;
}
```

### Distribution Example

Given 14,400 profiles and 80 VAs:

| Profile Index | VA Table | Position | Explanation |
|---------------|----------|----------|-------------|
| 0 | 1 | 1 | `(0 % 80) + 1 = 1`, `floor(0 / 80) + 1 = 1` |
| 1 | 2 | 1 | `(1 % 80) + 1 = 2`, `floor(1 / 80) + 1 = 1` |
| 79 | 80 | 1 | `(79 % 80) + 1 = 80`, `floor(79 / 80) + 1 = 1` |
| 80 | 1 | 2 | `(80 % 80) + 1 = 1`, `floor(80 / 80) + 1 = 2` |
| 14,399 | 80 | 180 | `(14399 % 80) + 1 = 80`, `floor(14399 / 80) + 1 = 180` |

<Check>
**Even Distribution Guarantee**: Each VA table receives exactly `profilesPerTable` profiles with no remainder.
</Check>

## Database Structure

### daily_assignments Table

Each distributed profile creates a record:

```sql
CREATE TABLE daily_assignments (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  campaign_id UUID REFERENCES campaigns(campaign_id),
  username TEXT,
  full_name TEXT,
  va_table_number INTEGER,  -- 1 to num_vas (typically 80)
  position INTEGER,         -- 1 to profiles_per_table (typically 180)
  job_id UUID,
  base_id TEXT,            -- Multi-tenant isolation
  status TEXT DEFAULT 'pending',
  assigned_at TIMESTAMP DEFAULT NOW()
);
```

## Querying Distributions

### Get All Assignments for a VA

```sql
SELECT username, full_name, position
FROM daily_assignments
WHERE campaign_id = 'campaign-123'
  AND va_table_number = 1
  AND base_id = 'base-789'
ORDER BY position ASC;
```

**Result**: 180 profiles assigned to VA Table 1, ordered by position (1-180).

### Count Profiles per VA

```sql
SELECT va_table_number, COUNT(*) as profile_count
FROM daily_assignments
WHERE campaign_id = 'campaign-123'
  AND base_id = 'base-789'
GROUP BY va_table_number
ORDER BY va_table_number ASC;
```

**Expected Result**: All VA tables show equal counts (e.g., 180 profiles each).

### Verify Distribution Balance

```sql
WITH distribution AS (
  SELECT va_table_number, COUNT(*) as count
  FROM daily_assignments
  WHERE campaign_id = 'campaign-123'
    AND base_id = 'base-789'
  GROUP BY va_table_number
)
SELECT 
  MIN(count) as min_profiles,
  MAX(count) as max_profiles,
  AVG(count) as avg_profiles
FROM distribution;
```

**Expected Result**: `min_profiles = max_profiles = avg_profiles = 180`

**Response**:
```json
{
  "success": true,
  "campaign_id": "campaign-123",
  "va_tables": 80,
  "profiles_per_table": 180,
  "total_distributed": 14400
}
```

**Backend Logic** (conceptual Python):
```python
@app.route('/api/distribute/<campaign_id>', methods=['POST'])
def distribute_profiles(campaign_id):
    base_id = request.headers.get('X-Base-ID')
    profiles_per_table = request.json.get('profiles_per_table', 180)
    
    # Get campaign details
    campaign = get_campaign(campaign_id, base_id)
    num_vas = campaign.num_vas
    
    # Get selected profiles for this campaign
    profiles = get_campaign_profiles(campaign_id, base_id)
    
    # Distribute profiles
    assignments = []
    for index, profile in enumerate(profiles):
        va_table = (index % num_vas) + 1
        position = (index // num_vas) + 1
        
        assignments.append({
            'campaign_id': campaign_id,
            'username': profile.username,
            'full_name': profile.full_name,
            'va_table_number': va_table,
            'position': position,
            'base_id': base_id
        })
    
    # Insert assignments into database
    supabase.table('daily_assignments').insert(assignments).execute()
    
    return jsonify({
        'success': True,
        'campaign_id': campaign_id,
        'va_tables': num_vas,
        'profiles_per_table': profiles_per_table,
        'total_distributed': len(assignments)
    })
```

## Airtable Table Structure

After distribution, profiles are synced to Airtable with the same structure:

### VA Table Naming Convention

- **Table 1** through **Table 80**
- Each table in the Airtable base corresponds to one `va_table_number`

### Airtable Columns

| Column Name | Type | Description |
|-------------|------|-------------|
| **ID** | Text | Instagram user ID |
| **Username** | Text | Instagram handle |
| **Full Name** | Text | Display name |
| **Position** | Number | Position within VA's list (1-180) |
| **Campaign Date** | Date | Date campaign was created |
| **Status** | Single Select | pending, contacted, completed |

### Sync Process

```typescript
// For each VA table (1-80)
for (let tableNum = 1; tableNum <= 80; tableNum++) {
  const assignments = getAssignmentsForTable(campaignId, tableNum);
  const tableName = `Table ${tableNum}`;
  
  const records = assignments.map(a => ({
    fields: {
      'ID': a.id,
      'Username': a.username,
      'Full Name': a.full_name,
      'Position': a.position,
      'Campaign Date': campaign.campaign_date
    }
  }));
  
  await airtable.base(baseId).table(tableName).create(records);
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Uneven distribution">
    **Problem**: Some VA tables have more profiles than others
    
    **Solutions**:
    1. Check if total profiles is evenly divisible by `num_vas`
    2. Verify round-robin algorithm is correctly implemented
    3. Query database to find unbalanced tables:
       ```sql
       SELECT va_table_number, COUNT(*) as count
       FROM daily_assignments
       WHERE campaign_id = 'campaign-123'
       GROUP BY va_table_number
       HAVING COUNT(*) != 180;
       ```
    4. Contact support if using standard 80×180 configuration
  </Accordion>
  
  <Accordion title="Missing VA tables">
    **Problem**: Distribution skips certain table numbers
    
    **Solutions**:
    1. Verify `num_vas` in job matches expected value:
       ```sql
       SELECT num_vas FROM scraping_jobs WHERE job_id = 'job-123';
       ```
    2. Check distribution logic isn't skipping indices
    3. Ensure no gaps in profile selection before distribution
  </Accordion>
  
  <Accordion title="Position numbers incorrect">
    **Problem**: Position values don't increment correctly (1, 2, 3...)
    
    **Solutions**:
    1. Verify position calculation: `floor(index / num_vas) + 1`
    2. Check that profiles are processed in order (index 0, 1, 2...)
    3. Query for position anomalies:
       ```sql
       SELECT va_table_number, MIN(position), MAX(position)
       FROM daily_assignments
       WHERE campaign_id = 'campaign-123'
       GROUP BY va_table_number;
       ```
  </Accordion>
  
  <Accordion title="Distribution failed error">
    **Problem**: Phase 2 fails with distribution error
    
    **Solutions**:
    1. Check backend logs for specific error messages
    2. Verify database connection is stable
    3. Ensure `campaign_id` was created successfully in Phase 1
    4. Check that profiles exist for the campaign:
       ```sql
       SELECT COUNT(*) FROM global_usernames
       WHERE used = true AND base_id = 'base-789';
       ```
  </Accordion>
</AccordionGroup>
